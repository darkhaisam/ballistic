Ammo DB (3NF) — База данных боеприпасов
Обзор проекта
Данный проект представляет собой структуру базы данных боеприпасов, нормализованную до третьей нормальной формы (3NF). Система предназначена для хранения, анализа и управления данными о пулях, патронах и справочных источниках, таких как Applied Ballistics, SAF, RAF и другие.

Особое внимание уделено требованиям Data Governance, включая версионирование, аудит изменений и контроль качества данных.

Основные сущности
Архитектура системы разделена на несколько ключевых блоков:


Пули (Bullet): Хранение физических (масса, диаметр, длина) и баллистических (BC G1/G7, форма оживала) характеристик.


Патроны (Cartridge): Описание готовых сборок, связывающее пули с конкретными типами пороха и гильз.


Порох (Powder): Справочник метательных веществ с указанием их плотности, энергии и скорости горения.


Метаданные (Metadata): Отдельные таблицы для аудита, контроля качества (quality flags) и уровней конфиденциальности.


Справочники (Lookup): Единая таблица для хранения всех доменных значений (калибры, материалы, производители и т.д.).

Архитектура и связи
Система построена на следующих принципах связности:


Bullet 1:N Cartridge: Одна пуля может использоваться во многих типах патронов.


Cartridge 1:N ReferenceData: Информация о патроне может подтверждаться данными из разных источников.


Полиморфные изображения: Таблица Image привязывается к объектам (bullet, cartridge, reference) через поля object_type и object_id.
+1


Metadata 1:1: Каждая запись пули или патрона сопровождается записью метаданных для обеспечения жизненного цикла данных.

Data Governance и аудит
Система реализует строгие правила управления данными:


Версионирование: Поле data_version отслеживает изменения в рамках жизненного цикла записи.


Качество: Статусы verified, draft, needs_review позволяют фильтровать надежные данные.


Безопасность: Поле confidentiality_level_id разграничивает доступ (публичный, ограниченный, конфиденциальный).


Соответствие стандартам: Поддержка тегов соответствия SAAMI, CIP и ISO.

Установка и развертывание
Для развертывания базы данных необходимо выполнить SQL-скрипт создания таблиц в следующем порядке:

Таблица Lookup и Source (базовые справочники).

Таблица Powder и Bullet.

Таблица Cartridge.

Таблицы метаданных и связей (ReferenceData, Image).
